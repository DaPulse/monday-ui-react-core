## Overview
Dropdowns present a list of options from which a user can select one option, or several.

## Do’s and Don’ts
<ComponentRules
  rules={[
    {
      positive: {
        component:
        <Dropdown
          defaultValue={[{ value: "English", label: "English" }]}
          options={[{ value: "English", label: "English" }, { value: "Hebrew", label: "Hebrew" }]}
          className="dropdown-stories-styles_rules-spacing"
        />,
        description:"Use dropdown to keep the user’s choice visible."
      },
      negative: {
        component:
          <Dropdown
            placeholder="Choose option" 
            options={[{ value: "English", label: "English" }, { value: "Hebrew", label: "Hebrew" }]}
            className="dropdown-stories-styles_rules-spacing"
          />,
        description:<>If you don’t want to keep the user’s choice visible, use the <Link href="/?path=/docs/inputs-menu--overview" withoutSpacing>Menu component</Link> instead.</>
      }
    },
    {
      positive: {
        component:
          <Dropdown
            placeholder="Placeholder text here" 
            options={[{ label: "Option 1" }, { label: "Option 2" }, { label: "Option 3" }]} 
            className="dropdown-stories-styles_rules-spacing"
          />,
        description:"Use the dropdown as a closed component. Users can only click on the items but not type or search."
      },
      negative: {
        component:
          <DialogContentContainer>
            <Combobox 
              placeholder="Search here"
              options={[ { id: "1", label: "Option 1" }, { id: "2", label: "Option 2" }, { id: "3", label: "Option 3" } ]}
            />
          </DialogContentContainer>,
        description:<>Don’t insert any fields inside a dropdown. If you need, use the <Link href="/?path=/docs/inputs-combobox--overview" withoutSpacing>Combobox</Link> instead.</>
      }
    }
  ]}

## Use cases and examples
### Dropdown with chips
Inside the advanced filters, a user can select multiple people from the dropdown menu, and they will be shown as Chips.
<Canvas>
  <Story name="Dropdown with chips">
    {() => {
      const options = useMemo(() => [
        { value: "Dor Yehuda", label: "Dor Yehuda", src: person1, type: Avatar.types.IMG, size: Avatar.sizes.SMALL, name: "Dor Yehuda", position: "?" },
        { value: "No", label: "Rotem Dekel", src: person1, type: Avatar.types.IMG, size: Avatar.sizes.SMALL, name: "Rotem Dekel", position: "(Product Designer)" },
        { value: "Yes", label: "Netta Muller", src: person1, type: Avatar.types.IMG, size: Avatar.sizes.SMALL, name: "Netta Muller", position: "(Brand Designer)" },
      ], []);
      return (
        <Dropdown
          defaultValue={[options[0]]}
          options={options}
          multi
          optionRenderer={OptionRenderer}
          className="dropdown-stories-styles_with-chips"
        />
      )
    }}
  </Story>
</Canvas>

### Dropdown inside a form
A classic dropdown presents options a user needs to choose from.
<Canvas>
  <Story name="Dropdown inside a form">

    }}
  </Story>
</Canvas>
